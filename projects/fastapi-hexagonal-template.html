FastAPI Hexagonal Template

Production-ready Python API template using hexagonal (clean) architecture — fully Dockerized, with automated tests, type safety, and CI/CD. Used as the foundation for multiple freelance AI projects.

🎯 Context & Problem

Most AI projects start with a notebook and end up as a messy FastAPI app with all logic in the route handler. When projects grow — adding agents, databases, external APIs — this monolithic approach becomes a maintenance nightmare.

This template solves that by enforcing hexagonal (ports & adapters) architecture from day one: business logic is completely isolated from infrastructure, making it easy to swap databases, add new adapters, or test in isolation.

Built from lessons learned across multiple freelance AI projects (MathildeCreations, ContentCrea), where this exact structure was used in production.

🏗️ Architecture

The structure follows the hexagonal (ports & adapters) pattern — the domain is the center, everything else is an adapter.

# Project structure src/ ├── domain/ │ ├── entities/ # Pure Python, no dependencies │ ├── use_cases/ # Business logic │ └── ports/ # Abstract interfaces ├── application/ │ ├── services/ # Orchestration layer │ └── dtos/ # Pydantic schemas └── infrastructure/ ├── api/ # FastAPI routers ├── repositories/ # DB adapters (SQLAlchemy) └── external/ # Third-party clients
  • Domain layer has zero external dependencies — pure Python classes with business rules, fully testable without a database or HTTP server
  • Ports are abstract interfaces (Python ABCs) that decouple business logic from infrastructure — swap PostgreSQL for MongoDB without touching use cases
  • FastAPI acts purely as a delivery mechanism — routes call application services, never domain logic directly
  • Dependency injection via FastAPI's DI system — adapters are injected at the router level
  • Docker Compose brings up the full stack (API + DB + Redis) with a single command
