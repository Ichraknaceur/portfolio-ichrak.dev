Content Agent System

End-to-end multi-agent AI system for automating professional online presence — Medium articles, LinkedIn scheduling, outreach automation, and AI monitoring, all with Human-in-the-Loop validation.

🐍 Python Streamlit Playwright Claude + GPT-4o-mini 🔒 Private — GitHub coming soon

Context

Building a strong AI presence on LinkedIn and Medium takes hours each week — writing articles, crafting posts, finding relevant people to connect with, and tracking the latest research. This project automates the entire workflow through specialized LLM agents, each handling one well-defined task.

The key design principle is Human-in-the-Loop: every critical action (publishing a post, sending a connection request) goes through a manual review step in the Streamlit dashboard before anything is executed. The agents generate and suggest; the human decides and approves.

4-Phase Pipeline

Phase 1
✍️ Medium Article Generation
Full article pipeline: draft → SEO optimization → quality scoring → revision loop. Claude produces long-form, structured technical articles.
Phase 2
📅 LinkedIn Scheduling
Weekly editorial calendar with post generation, manual approval UI, and Playwright-based auto-publishing to LinkedIn.
Phase 3
🤝 LinkedIn Outreach
Profile search → enrichment → LLM-generated personalized connection notes (≤300 chars) → review → automated sending. Max 15/day limit enforced.
Phase 4
📡 AI Monitoring (Veille)
RSS feeds from 7 curated AI sources (TDS, HuggingFace, The Batch, OpenAI…). LLM summaries + LinkedIn post suggestions sent directly to the scheduler.

Human-in-the-Loop Design

🔐
No action is taken without human approval. Every generated post, connection note, and article passes through a review step in the Streamlit dashboard. The LLM suggests; the human decides. This prevents automation from damaging professional reputation and keeps content authentically personal.

The outreach module also enforces a hard cap of 15 connections per day to comply with LinkedIn's limits and prevent account restrictions. Acceptance tracking is built in — a background check compares connection degree badges to detect accepted requests.

Agent Architecture

All agents inherit from BaseAgent which implements a provider-agnostic agentic loop with tool calls. The LLM provider (Anthropic or OpenAI) can be switched dynamically from the UI sidebar without restarting the app.

Agent Role Notes
BaseAgent Agentic loop core Multi-provider (Anthropic / OpenAI), tool call dispatch, provider cache
MediumAgent Article drafting Generates full technical article: intro, body, conclusion in Markdown
SEOAgent SEO optimization Keywords, meta description, heading structure, internal links
QAJudgeAgent Quality scoring Scores article 0–10, flags issues. Triggers revision if score < threshold
RevisionAgent Article revision Improves article based on QA feedback (max 2 revision loops)
OutreachAgent Connection notes Personalized LinkedIn notes ≤ 300 chars, context-injected from sidebar
VeilleAgent AI monitoring French summaries (~150 words) + engaging LinkedIn post suggestions

Playwright Automation Layer

LinkedIn interaction is handled by a linkedin_scraper.py module running Playwright in non-headless mode — LinkedIn is significantly better at detecting headless browsers, so visible Chrome is used for scraping, posting, and sending connections.

Anti-detection measures include human-like delays between actions, webdriver attribute spoofing, and persistent session cookies stored in data/linkedin_cookies.json (login required only once per session).

The linkedin_poster.py module handles the full LinkedIn post publishing flow via browser automation, while check_acceptances() visits each "sent" connection profile and detects if the degree badge changed to 1st — confirming acceptance without any LinkedIn API access.
