LLM Content Agent

Multi-task agentic system that transforms raw notes into polished LinkedIn posts and Medium articles — with LangGraph orchestration, quality evaluation, and a critique-refine loop.

Personal Project LangGraph Claude API Multi-Agent Python
⌥ View on GitHub

🎯 Context & Problem

AI engineers and researchers produce a lot of insights — in notes, papers, experiments, and meetings. But turning raw knowledge into polished, audience-ready content (LinkedIn posts, technical articles) is time-consuming and requires a different writing mode.

This project automates that pipeline: given rough notes or a topic, the agent produces platform-optimized content, critiques it, and refines it — all without human intervention in the loop.

🏗️ Technical Architecture

The system is a LangGraph stateful graph with specialized nodes — each responsible for one stage of the content pipeline.

Raw Notes / Topic
→
Planner Agent
→
Content Brief
↓
LinkedIn Writer
+
Medium Writer
→
Draft Content
↓
Critic Agent (Claude)
→
Quality Score + Feedback
↓ (if score < threshold)
Refiner Agent
→
Final Content
  • LangGraph models the full pipeline as a directed graph with conditional edges — enabling branching (refine or publish) based on quality scores
  • Specialized writer nodes use platform-aware prompts: LinkedIn posts optimized for engagement, Medium articles for technical depth
  • Claude API as Critic scores content on clarity, accuracy, engagement, and platform fit — returning structured feedback
  • Iterative refinement loop runs up to 3 cycles, with each pass addressing the critic's specific objections
